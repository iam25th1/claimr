What changed and why

/project (overview)
  - Top: PageHeader with eyebrow + title + "Post a job" action button on
    the right. Action is now where the user expects it, not stuck at the
    bottom as PostJobCTA was.
  - Sections each have their own SectionHeader with optional "See all" link
  - Order: hero (escrow breathing number) -> wallet -> stats -> verifier
    insight -> recent activity -> active jobs preview
  - Removes the bottom PostJobCTA component (action moved to the header)

/project/post (formerly a button-label flow)
  - 4-step visible stepper: Details -> Criteria + pay -> Visibility -> Review + lock
  - StepBar at the top shows progress with done/active/upcoming states
  - Review step shows a complete summary before the lock action
  - During the on-chain action, an explicit "Approving USDC -> Locking in escrow"
    progress panel renders below the summary. No more guessing what the
    button label means
  - Errors use the friendly ErrorCallout (insufficient funds gets "Fund wallet"
    button, indexing latency gets refresh, etc) instead of raw chain errors

/project/jobs (formerly an inline list)
  - Proper table on desktop: ID, Job, Claimant, USDC, Status, ChevronRight
  - Card layout on mobile (auto-collapse via responsive grid)
  - Search box at the top: matches title, criteria, job id
  - Filter pills: All / Open / Claimed / Under review / Paid
  - Rows have layout animation when filter changes
  - Each row links to /project/jobs/[id] (the detail page shipped in admin batch)
  - Loading shows JobCardSkeleton x3
  - Empty state copy adapts: "No jobs posted yet" with Post-a-Job CTA when
    truly empty, vs "No [filter] jobs" with no CTA when just filtered out
