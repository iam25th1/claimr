How browse mode actually works

  - Anyone (no sign-in) can hit /, /dashboard/discover, navigate the
    sidebar, see the Featured and Latest job sections, read all listings.
  - Clicking Claim on any job - if not authenticated - redirects to
    /onboarding?role=creator. That redirect was already in place; no
    change needed.
  - Visiting /dashboard/my-jobs as a guest renders the page normally but
    with the GuestBanner pinned at the top. The list below is empty
    (since no wallet is connected). Same for wallet, earnings.
  - Sidebar shows lock icons on My Jobs, Earnings, Wallet, Settings
    when signed out. Discover is the only unlocked item.
  - Bottom of sidebar shows a Sign In button instead of the user profile
    avatar.


Platform jobs - how the tag works

  No on-chain change. The marketplace components (featured-jobs,
  latest-jobs) read NEXT_PUBLIC_ADMIN_WALLETS via lib/admin-jobs.ts and
  compare each job's project address against the allowlist. Any match
  gets a "Platform" badge.

  When an admin uses /hq/c8m4z2/post to create a job, the job posts on
  chain from the admin wallet. Since the admin wallet is in the
  allowlist, the badge appears automatically. Zero metadata storage.

  Side effect: any non-admin job posted FROM an admin wallet would also
  be tagged Platform. Since admins post directly with their wallet, this
  is consistent.


Vercel env var format reminder

  NEXT_PUBLIC_ADMIN_WALLETS=0xaaa...,0xbbb...,0xccc...
  ADMIN_EMAILS=alice@example.com,bob@example.com

  Comma-separated. Case and surrounding spaces don't matter (the parser
  trims and lowercases). Add as many as you want.

Test after deploy

  Open an incognito window. You should be able to:
    - Hit / and see Launch App as the primary CTA
    - Click Launch App, land on /dashboard/discover, see all jobs
