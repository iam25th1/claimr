Auth fix and sign-in/sign-up split

Three things in this batch.

1. Diagnose and fix the 500 on signup

   Most likely cause: CIRCLE_API_KEY missing in Vercel env vars. Step one
   is verify in Vercel dashboard - Settings - Environment Variables that
   both CIRCLE_API_KEY and NEXT_PUBLIC_CIRCLE_APP_ID are set on all
   environments (Production, Preview, Development).

   The onboard route now also detects this case directly and returns
   "CIRCLE_API_KEY is not set in Vercel env vars" instead of "Internal
   onboarding error", so the UI shows the real reason on the screen.

2. Real bug fix in the auth flow

   When an existing user tries to sign in via the same flow, Circle does
   not return a challengeId (PIN already exists). The old code called
   executeChallenge(null) which threw, surfacing as a generic error to
   the user. Now:
     - The onboard route returns alreadyInitialized=true in that case
     - providers.tsx skips executeChallenge when alreadyInitialized is set
     - Goes straight to finalize, which looks up the existing wallet

   Also handles Circle error codes 155206 (pinAlreadySet) and 155709
   (walletAlreadyExists) as "already initialized" cases.

3. Sign in vs sign up split

   /onboarding now reads ?mode= from the URL:
     mode=signup (default)   full flow: role -> email -> verify -> wallet -> profile
     mode=signin             short flow: email -> verify -> redirect

   Visible mode toggle at the top of /onboarding lets the user switch.

   Routing changes:
     - Navbar "Sign in" (signed-out state)  -> /onboarding?mode=signin
     - Landing hero "I'm a Creator"         -> /onboarding?role=creator&mode=signup
     - Landing hero "I'm a Project"         -> /onboarding?role=project&mode=signup

   The sign-in flow skips the role picker, wallet-ready, and profile steps
   since returning users already have all of those. Just email, OTP, in.
