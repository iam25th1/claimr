Two env vars to set in Vercel

  NEXT_PUBLIC_ADMIN_WALLETS   client-side: comma-separated lowercased 0x addresses
                              that should see the management UI. UI-only gating.

  ADMIN_EMAILS                server-side: comma-separated lowercased emails that
                              are allowed to call the approve/reject/activity APIs.
                              THIS is the real gate. Even if someone DOM-hacks past
                              the UI, the API rejects them without an allowed email.

Set both to your wallet + signup email. Get your wallet from the wallet
card on /project or /dashboard/wallet.

If you already have VERIFIER_PRIVATE_KEY set (for the AI verifier), that
same key signs the manual approve/reject. No new key needed.


URL obscurity

The slug "/hq/c8m4z2" is hardcoded as a folder name AND in lib/admin-config.ts.
If you want to rotate it, you have to:
  1. Rename the folder app/hq/c8m4z2 to app/hq/<new-slug>
  2. Update ADMIN_BASE_PATH in lib/admin-config.ts to match
The sidebar and layout pick up the constant automatically.
