CRITICAL: set the admin allowlist before testing
------------------------------------------------

Without NEXT_PUBLIC_ADMIN_WALLETS set, every signed-in user gets redirected
out of /admin. Set yourself first.

1. In `.env.local` at the repo root, add:

      NEXT_PUBLIC_ADMIN_WALLETS=0xYOUR_WALLET_HERE

   Comma-separated to allow multiple wallets. Lowercased or mixed-case
   both work.

2. In Vercel project env (Settings -> Environment Variables), add the same
   var for Production, Preview, and Development.

Negative test (gate works)
---------------------------

Sign in with a wallet that's NOT in NEXT_PUBLIC_ADMIN_WALLETS, then visit
/admin or any sub-route. You should briefly see the "Admin access only"
panel then redirect to /dashboard.


What's NOT in Phase A (deferred to Phase B)
--------------------------------------------

- Write actions (manual approve/reject, refund stuck escrow, edit verifier
  prompt at runtime, pause/unpause, ban wallets)
- Server-side gate on /api/admin/* endpoints (the read-only ones don't
  need it; writes will)
- Persisted verifier reasoning (currently in-memory Map that resets on
  Lambda cold-start; Phase B = Vercel KV)
- Full chain tx history (current "activity" is verifier decisions +
  derived state changes from useJobs; real indexer is Phase B)
