Motion wiring patch — Batch 1 retrofit
=======================================

What this does
--------------
Wires the Batch 0 primitives (AnimatedNumber, StatePill) into the four
dashboard surfaces that decide demo impact:

  components/claimr/hero-escrow-number.tsx
    - container now breathes (subtle 3.6s scale loop, 0.6% amplitude)
    - escrow total counts up smoothly via AnimatedNumber

  components/claimr/project-job-cards.tsx
    - StatePill replaces the inline status span
    - cards now layout-animate (slide in/out, reflow on status change)
    - drops the now-unused STATUS_LABELS + STATUS_STYLES tables

  app/project/jobs/page.tsx
    - StatePill replaces the inline status span
    - cards layout-animate inside an AnimatePresence

  components/claimr/my-jobs-list.tsx
    - StatePill (size sm) added next to each job title
    - cards layout-animate inside an AnimatePresence

Order-independent with the previous empty-states patch (claimr-batch-4.zip).
Can be applied before or after. Verified both orderings produce the same
final state and type-check clean.


Apply
-----

1. Zip lands at:
   C:\Users\Bright\Documents\GitHub\claimr-motion-wiring.zip

2. Extract into the repo (just the apply script, no new source files):

   cd C:\Users\Bright\Documents\GitHub ; Expand-Archive -Force claimr-motion-wiring.zip -DestinationPath claimr

   If git status is clean after, use File Explorer extract instead.

3. Run from the repo root:

   cd C:\Users\Bright\Documents\GitHub\claimr ; node apply-motion-wiring.js

   Script self-deletes on success. Idempotent on re-run.

4. Verify in dev:

   npm run dev

   /project           hero number breathes, counts up on first paint,
                      counts again when you post or pay out a job
   /project/jobs      StatePill on each row, smooth entrance, reflow when
                      filter changes
   /dashboard/my-jobs StatePill appears next to title in each tab

5. Commit + push:

   git add -A ; git commit -m "batch 1 retrofit: wire AnimatedNumber + StatePill into dashboard surfaces" ; git push https://iam25th1@github.com/iam25th1/claimr.git


What this does NOT do
---------------------

- /project/escrow USDC fly-to-creator animation. That's the bigger Batch 1
  moment piece and gets its own batch.
- Living background. The primitive exists but the call to wire it into
  /project is deferred until we've seen how the dashboard feels with the
  number breathing and the cards animating. May not need it.
- Drop ignoreBuildErrors: true from next.config.mjs. Still open.
