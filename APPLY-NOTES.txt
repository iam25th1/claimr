Tour fix v2
================================

What broke

  1) Step 6 targeted the navbar "?" icon, which has `hidden sm:inline-flex`
     and is display:none on mobile. The poller waited 3.2s for non-zero
     dimensions, never found them, then rendered the fallback centered
     tooltip.

  2) Viewport state initialized to {1024, 768} (the SSR default). On
     mobile devices the centered fallback tooltip computed an offscreen
     left position based on those wrong dims. The X/Finish buttons
     ended up off the right edge of the screen, so the user could see
     a dim overlay but no way to dismiss it.

  3) The tour only had 6 steps and skipped meaningful surfaces like the
     sign-in CTA at the bottom of the sidebar.


What this batch ships

  Viewport state now reads window.innerWidth/innerHeight via a useState
    function initializer on first client render. Mobile devices get
    the correct dims immediately so the fallback tooltip lands within
    the visible viewport. Plus a centeredTooltipPos helper that clamps
    to viewport boundaries no matter what.

  Step 6 target moved.
    Old target: [data-tour-id="tour-replay"]   (navbar ? icon)
    New target: the on-page "Take a tour" button which has no responsive
    hiding and is always measurable.

  Tour now has 8 steps, covering everything user-visible:
    1. discover-header  - welcome to Discover
    2. discover-search  - search and filter
    3. discover-featured - featured jobs and badges
    4. discover-latest  - latest jobs + how to claim
    5. dashboard-sidebar - the navigation overview
    6. sidebar-cta      - NEW. The bottom of the sidebar where the
                          sign-in CTA lives. Three div returns in
                          dashboard-sidebar.tsx now have
                          data-tour-id="sidebar-cta" so this works
                          for both guest mode and signed-in users
    7. tour-replay      - the navbar ? icon as a tip
    8. tour-trigger     - the on-page Take a tour button (replay CTA)

  Tooltip card adds maxWidth: calc(100vw - 24px) so it can never spill
    off either edge on narrow screens.

  When a step has no measurable target the previous step's rect now
    clears so a stale spotlight from step N-1 doesn't bleed into step N.
