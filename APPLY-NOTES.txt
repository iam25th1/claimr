Slots on post job

  components/claimr/post-job-stepper.tsx           PATCHED

  Step 2 (Criteria + pay) now has a "How many creators can take this job?"
  counter (1 - 10, default 1). Each slot is independently claimable by a
  different creator.

  Step 4 (Review) shows the total escrow (pay x slots) and a heads-up that
  multi-slot posts require N+1 PIN entries (1 approve + N postJob calls).

  Lock flow:
    1. Approve total = pay x slots in one transaction
    2. Post N separate jobs on chain, each locking pay USDC, all using
       the same title and criteria
    3. Progress UI shows "Posting job 1 of N", "Posting job 2 of N", etc.

  Contract reality: the on-chain contract is one-claimer-per-job. This is
  a UI workaround that creates N siblings - same title, same criteria,
  separate jobs that creators claim independently. To the user it feels
  like a multi-slot job. On chain it's N posts.

Honest swap rate

  components/claimr/swap-card.tsx                  REBUILT

  Removed the misleading hardcoded "estimatedOut = amount * 0.918" line.
  The 0.918 rate was a UI fiction - the actual rate is whatever the
  Starlight pool decides at swap time, which on testnet can be very far
  from 1:1.

  New behaviour:
    - The "To" panel says "Rate set by liquidity pool" instead of showing
      a fake number
    - A disclaimer below explains rates are determined at swap time and
      testnet rates can vary significantly
    - After the swap completes, the destination balance is refetched and
      diffed against the pre-swap value, then the actual amount received
      is shown in a success banner for 8 seconds

  This means the user sees the true number ("You received 1.20 EURC")
  instead of a misleading "1.20 EURC where 18.36 was implied".
